Readonly microcontrollerFactory

microcontrollerFactory: MicrocontrollerFactory<SloMappingSpec<any, any, any>, ServiceLevelObjective<any, any, any>>
  • Defined in libs/core/src/lib/slo/public/control/slo-control-loop.ts:69

This factory is used to instantiate a new ServiceLevelObjective when an SLO mapping is received. Before starting the control loop, factories for all watched SLO mapping kinds must be registered.

important

The factory should only instantiate a ServiceLevelObjective. The factory MUST NOT call the ServiceLevelObjective.configure() method.

Readonly watchHandler

watchHandler: SloWatchEventsHandler
  • Defined in libs/core/src/lib/slo/public/control/slo-control-loop.ts:60

The WatchEventsHandler that can be used to connect this control loop to an ObjectKindWatcher.

note

This property is only set if the control loop is currently active.

addSlo

  • addSlo(key: string, sloMapping: SloMapping<any, any, any>): Promise<ServiceLevelObjective<any, any, any>>
    • Defined in libs/core/src/lib/slo/public/control/slo-control-loop.ts:84

    Creates a new SLO instance using the specified sloMapping and adds that instance under the key, such that it will be evaluated on every loop iteration, starting with the next one.

    If an SLO with the same key already exists, it will be replaced.

    Parameters

    • key: string

      The key that should be used to identify the SLO. This must be unique within the cluster.

    • sloMapping: SloMapping<any, any, any>

      The SloMapping that describes the SLO.

    Returns Promise<ServiceLevelObjective<any, any, any>>

    A Promise that resolves to the created ServiceLevelObjective object or rejects if an error occurs.
